# 시스템 콜
**시스템 콜**(**System Call**)은 사용자 프로그램이 운영체제의 커널이 제공하는 서비스를 안전하게 사용할 수 있도록 도와주는 인터페이스이다.
시스템 콜 호출 시 CPU가 커널 모드로 전환되어 작업을 수행한 뒤 다시 사용자 모드로 돌아온다.

## 시스템 콜 유형
- **프로세스 제어**(**process control**)
  - 프로세스의 생성, 종료, 실행, 중지 및 속성 관리 작업을 수행한다.
- **파일 조작**(**file manipulation**)
  - 파일의 생성, 삭제, 열기, 닫기, 읽기, 쓰기 작업을 수행한다.
- **장치 관리**(**device management**)
  - 하드웨어 장치를 제어하고 상태 정보를 읽거나 설정한다. 
- **정보 유지**(**information maintenance**)
  - 시스템 날짜/시간, 프로세스 정보 등 시스템 데이터를 조회하거나 설정한다.
- **통신**(**communication**)
  - 프로세스 간 데이터를 교환한다.
- **보호**(**protection**)
  - 시스템 자원에 대한 접근 권한을 제어한다.

| | Unix | Windows |
|------|---|---|
| **프로세스 제어** | `fork()`<br>`exit()`<br>`wait()` | `CreateProcess()`<br>`ExitProcess()`<br>`WaitForSingleObject()` |
| **파일 조작** | `open()`<br>`read()`<br>`write()`<br>`close()` | `CreateFile()`<br>`ReadFile()`<br>`WriteFile()`<br>`CloseHandle()` |
| **장치 관리** | `ioctl()`<br>`read()`<br>`write()` | `SetConsoleMode()`<br>`ReadConsole()`<br>`WriteConsole()` |
| **정보 유지** | `getpid()`<br>`alarm()`<br>`sleep()` | `GetCurrentProcessID()`<br>`SetTimer()`<br>`Sleep()` |
| **통신** | `pipe()`<br>`shmget()`<br>`mmap()` | `CreatePipe()`<br>`CreateFileMapping()`<br>`MapViewOnFile()` |
| **보호** | `chmod()`<br>`umask()`<br>`chown()` | `SetFileSecurity()`<br>`InitializeSecurityDescriptor()`<br>`SetSecurityDescriptorGroup()` |
