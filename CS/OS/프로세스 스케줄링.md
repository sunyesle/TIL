# 프로세스 스케줄링

**스케줄링**은 CPU를 어떤 프로세스에게 할당해 줄지 결정하는 작업이다. CPU에서 프로세스를 할당하고 회수함으로써 CPU 이용률을 높이고, 사용자에게 허용 가능한 응답 시간을 제공하는 것을 목표로 한다.

## 스케줄링 큐
운영체제가 이해하는 프로세스는 PCB이기 때문에 각 큐는 PCB의 연결 리스트 형태로 관리된다.

- **작업 큐**(**Job Queue**): 시스템에 존재하는 모든 프로세스들
- **준비 큐**(**Ready Queue**): CPU 할당을 기다리는 준비 상태의 프로세스들이 대기하는 큐
- **장치 큐**(**Device Queue**): 입출력 작업을 기다리는 프로세스들이 대기하는 큐로, 일반적으로 각 장치마다 별도의 큐가 존재한다.

### 장기 스케줄러
> 어떤 프로세스를 준비 큐에 넣을지 결정한다.

**장기 스케줄러는 메모리에 동시에 올라와 있는 프로세스의 수를 조절하는 역할을 한다.** **작업 스케줄러**라고도 부른다.

디스크에서 프로그램을 가져와 커널에 등록하면 프로세스가 되는데, 이때 어느 프로그램을 가져와 커널에 등록할지 결정한다.

장기 스케줄러는 드물게 호출되기 때문에 상대적으로 속도가 느린 것이 허용된다.

현대의 시분할 시스템에서는 장기 스케줄러의 역할이 크게 감소하였다. 과거 자원이 부족하던 시절에는 제한된 메모리에 너무 많은 프로세스를 동시에 적재하면 각 프로세스에 할당되는 메모리 양이 줄어들어 시스템의 효율이 떨어졌기 때문에, 장기 스케줄러가 이를 조절하는 역할을 수행했다.

### 중기 스케줄러
> 메모리에 적재된 프로세스 수를 관리한다.

**중기 스케줄러는 스와핑을 통해 메모리에 적재된 프로세스 수를 조절하는 역할을 한다.** **스와퍼**(Swapper)라고도 부른다.

메모리의 여유 공간을 확보하기 위해 일부 프로세스의 메모리를 디스크의 스왑 영역으로 옮기고(swap-out), 이후 메모리에 여유가 생기면 다시 적재한다.(swap-in)

중기 스케줄러의 등장으로 프로세스의 상태에는 중지 상태가 추가되었다.(suspended ready, suspended blocked)

현대의 시분할 시스템에서는 장기 스케줄러의 역할이 축소되어, 프로세스가 시작 상태가 되면 곧바로 메모리에 적재되는 경우가 많다.
이때 과도한 프로세스가 존재하여 메모리 압박이 발생하면, 중기 스케줄러가 프로세스의 수를 조절한다.

### 단기 스케줄러
> 준비 상태의 프로세스 중에서 어떤 프로세스에게 CPU를 할당할지 결정한다.

**단기 스케줄러는 준비 상태의 프로세스 중에서 어떤 프로세스에게 CPU를 할당할지 결정하는 역할을 한다.** **CPU 스케줄러**라고도 부른다.

타이머 인터럽트, I/O 작업이 완료될 때, 시스템 콜 등 다양한 이벤트가 발생할 때마다 호출되어 다음에 실행할 프로세스를 선택한다.

단기 스케줄러는 매우 빈번하게 호출되므로 수행 속도가 충분히 빨라야 한다.

## 컨텍스트 스위칭(Context Switching)
프로세스가 CPU를 사용하다가 중단될 때, 이후 다시 실행되기 위해 현재 실행 상태를 저장해야 한다. 이를 **컨텍스트**라고 하며, 프로그램 카운터(PC), CPU 레지스터들의 값, 메모리 정보 등이 포함된다.

**컨텍스트 스위칭**은 현재 실행 중인 프로세스의 컨텍스트를 PCB에 저장하고, 다음에 실행할 프로세스의 컨텍스트를 PCB로부터 복원하여 CPU에서 실행하도록 하는 과정을 말한다.

### 컨텍스트 스위칭 오버헤드
컨텍스트 스위칭이 너무 잦아지면 오버헤드가 발생해 성능이 떨어질 수 있다.

- **캐시 무효화**
  - 컨텍스트 스위칭 발생 시 cache memory flush가 이뤄진다. 기존 프로세스의 캐시 데이터는 무효화되고 새로운 프로세스의 데이터가 캐시에 로드된다. 이로 인해 캐시 미스가 발생하며, 메모리에서 CPU로 데이터를 가져와야 하므로 메모리 접근 시간이 증가하게 돼서 성능 저하가 발생할 수 있다.
- **스케줄러 오버헤드**
  - 운영체제의 스케줄러가 어떤 프로세스를 실행할지 결정하는데 소요되는 시간과 자원도 오버헤드에 포함된다.
- **레지스터 저장 및 복원**
  - 현재 실행 중인 프로세스의 레지스터 상태를 저장하고 새로 실행될 프로세스의 레지스터 상태를 복원하는 작업이 필요하다.
- **메모리 관리 오버헤드**
  - 새로운 프로세스의 페이지 테이블과 관련된 작업이 필요하다.
