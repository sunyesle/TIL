# 프로세스 제어 블록(PCB, Process Control Block)
CPU는 한 번에 하나의 프로세스만 실행할 수 있다.
그러나 운영체제는 CPU 스케줄링을 통해 여러 프로세스를 번갈아 실행함으로써 다수의 프로세스가 동시에 실행되는 것처럼 보이게 한다.

이때 운영체제가 프로세스를 관리하고 제어하기 위해서는 프로세스의 식별자, 상태 등의 메타데이터를 저장해놓아야 한다.
이러한 정보를 담고 있는 데이터 구조가 **PCB**다.

PCB는 프로세스 생성과 동시에 커널 영역에 생성되며, 프로세스가 종료되면 PCB 또한 제거된다.

## PCB 구조
운영체제에 따라 다를 수 있지만 일반적으로 PCB에는 다음과 같은 정보가 포함된다.
- **프로세스 ID**: 프로세스 ID(PID), 부모 프로세스 ID(PPID)
- **프로세스 상태**: 생성, 준비, 실행, 대기 등 상태
- **프로그램 카운터**(**PC**): 다음에 실행할 명령어의 주소
- **CPU 레지스터**: CPU 레지스터 관련 정보
- **CPU 스케줄링 정보**: 우선순위 정보 및 스케줄링 큐에 대한 포인터 등
- **메모리 관리 정보**: 페이지 테이블 또는 세그먼트 테이블
- **회계 정보**: 사용자 및 커널 CPU 사용 시간, 계정 번호, 한도 등
- **I/O 상태 정보**: 할당된 장치, 열린 파일 목록 등

## 프로세스 테이블
운영체제는 각 프로세스의 PCB를 효율적으로 관리하기 위해 프로세스 테이블을 사용한다.
프로세스 테이블에는 PCB의 포인터가 저장되며, 연결 리스트로 관리되어 삽입/삭제가 용이하다.

이를 통해 운영체제는 특정 프로세스의 PCB에 빠르게 접근할 수 있다.
