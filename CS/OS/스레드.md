# 스레드
## 프로세스와 스레드
**프로세스**(Process)는 프로그램의 실행 중인 인스턴스로, 독립적인 주소 공간과 자원을 할당받는 실행 단위이다.

**스레드**(Thread)는 프로세스 내에서 실행되는 흐름의 단위로, 동일한 프로세스의 주소 공간과 자원을 공유하며 독립적으로 실행된다.

### 차이점
|  | **프로세스** | **스레드** |
|---|---|---|
| **정의**  | 컴퓨터에서 실행되는 프로그램의 인스턴스이다. | 프로세스 내에서 실행되는 흐름의 단위이다. |
| **메모리 구조**      | 독립적인 주소 공간과 자원을 가진다. | 같은 프로세스 내에서 코드, 데이터, 힙 영역과 자원을 공유하고, 각 스레드는 독립적인 스택을 가진다. |
| **생성 비용**    | 주소 공간 초기화 및 자원 할당이 필요해 상대적으로 비용이 크다. | 기존 프로세스의 자원을 공유하므로 생성 비용이 낮고 빠르다. |
| **통신 방식** | IPC는 커널을 통한 데이터 전달과 복사 과정이 필요해 상대적으로 느리다. | 스레드 간 통신은 동일한 주소 공간을 공유하므로 빠르다. |
| **독립성**     | 각 프로세스는 서로 독립적으로 실행된다. | 이를 소유한 프로세스에 의해 제어된다. 프로세스 내의 모든 스레드는 프로세스 변경으로 인해 영향을 받을 수 있다. |
| **오버헤드**    | 별도의 메모리 및 자원 관리로 인해 오버헤드가 크다. | 자원을 공유하므로 오버헤드가 상대적으로 작다. |
| **사용 목적**    | 격리된 실행, 안정성 및 보안이 필요한 작업에 적합하다. | 병렬 처리 및 응답성 향상이 필요한 작업에 적합하다. |
| **장애 영향** | 하나의 프로세스 오류는 다른 프로세스에 영향을 주지 않는다. | 하나의 스레드 오류가 전체 프로세스에 영향을 줄 수 있다. |

## 스레드의 구성 요소
각 스레드는 자체 프로그램 카운터, 레지스터 값, 스택 영역을 갖지만, 동일한 프로세스 내에서 코드/데이터/힙 영역, 열린 파일과 같은 자원들을 공유한다.
