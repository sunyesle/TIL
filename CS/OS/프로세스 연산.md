# 프로세스 연산

## 프로세스 계층 구조
각 프로세스에는 프로세스 ID(PID)라고 하는 정수 식별자가 부여된다. 또한 각 프로세스는 부모 프로세스 ID(PPID)를 가진다.

일반적인 UNIX 시스템에서 PID 0은 커널이 생성하는 특수한 프로세스로, 시스템 초기화 과정 일부를 수행한 이후에는 CPU 코어가 다른 작업을 수행하지 않을 때마다 실행되는 idle 태스크로 동작한다.

시스템 시작 시 PID 0은 init 프로세스를 생성하며, init 프로세스에는 PID 1이 부여된다. init 프로세스는 모든 시스템 데몬과 사용자 로그인을 실행하고, 다른 모든 프로세스의 최상위 부모 프로세스가 된다.

## 프로세스의 생성과 종료
프로세스는 `fork()` 시스템콜을 통해 자식 프로세스를 생성하고, `exec()` 시스템콜을 호출하여 자식 프로세스의 메모리를 새로운 프로그램으로 덮어씌운다.

부모 프로세스는 `wait()` 시스템콜을 호출하여 자식 프로세스가 종료될 때까지 대기한다.

자식 프로세스가 `exit()` 시스템콜을 호출하면 대부분의 자원(메모리, 파일 등)은 커널에 의해 해제되지만, 종료 상태는 부모에게 전달되기 위해 프로세스 테이블에 유지된다.

부모 프로세스는 `wait()`을 통해 종료 상태를 수거하고, 그 시점에 자식 프로세스의 정보가 프로세스 테이블에서 완전히 제거된다.
